# Action Types

The following tables list all available actions that can be added to an activity.

## Pre-Session (Grant) Actions

| Action Type                      | Action Description                                                                                                                                                                                                                                                                                                       | Related Fields                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Add ADUC Permissions             | Add AD rights to the login account                                                                                                                                                                                                                                                                                       | <ul><li>AD Object Type — Select which type of object to manage in the Organizational Unit, a User or Computer.</li><li>AD Organizational Unit — Enter a pre-existing Organizational Unit the login account will have delegated permissions for.</li><li>AD Rights to be Added — Select which right will be delegated to the login account for managing the previously specified Organizational Unit</li><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li><li>Paired Action's Name — (Optional) Edit the name of the paired action</li></ul> |
| Add Microsoft Entra ID Role      | Add login account to an Microsoft Entra ID role                                                                                                                                                                                                                                                                          | <ul><li>Microsoft Entra ID Role (Allows Custom Entries) — Enter which Microsoft Entra ID Directory Role will be added to the login account, or select from the Role options in the dropdown.</li><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li><li>Paired Action's Name — (Optional) Edit the name of the paired action</li></ul>                                                                                                                                                                                                        |
| Add Sudoers Rule                 | Create a new rule file in sudoers.d directory to grant user additional permissions                                                                                                                                                                                                                                       | <ul><li>Run As — Enter which account(s) the logged-in user can run commands as</li><li>Commands — Enter which commands the user can run as a comma-separated list, or enter "ALL" for all commands on the target resource</li><li>Prompt for Password — If enabled, the logged-in user will need to enter their password to run elevated commands</li><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li><li>Paired Action's Name — (Optional) Edit the name of the paired action</li></ul>                                                   |
| Add to Microsoft Entra ID Group  | Add login account to a Microsoft Entra ID Security Group                                                                                                                                                                                                                                                                 | <ul><li>Microsoft Entra ID Security Group (Allows Custom Entries) — Enter which Microsoft Entra ID Security Group will be added to the login account</li><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li><li>Paired Action's Name — (Optional) Edit the name of the paired action</li></ul>                                                                                                                                                                                                                                                |
| Add User to Domain Group         | Add login account to a domain group                                                                                                                                                                                                                                                                                      | <ul><li>Domain — Enter an Active Directory domain</li><li>Group (Allows Custom Entries) — Enter a pre-existing Group the login account will be added to in the specified Domain</li><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li><li>Paired Action's Name — (Optional) Edit the name of the paired action</li></ul>                                                                                                                                                                                                                     |
| Add User to Local Group          | Add login account to a local group                                                                                                                                                                                                                                                                                       | <ul><li>Group (Allows Custom Entries) — Enter a pre-existing Group the login account will be added to in the specified Domain</li><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li><li>Paired Action's Name — (Optional) Edit the name of the paired action</li></ul>                                                                                                                                                                                                                                                                       |
| Add Requester to Database Server | Add login account to the database server                                                                                                                                                                                                                                                                                 | <ul><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li><li>Paired Action's Name — (Optional) Edit the name of the paired action</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                     |
| Add User to Database Role        | Add login account the Database Role.                                                                                                                                                                                                                                                                                     | <ul><li>Database – (Optional) If a database is specified, the role gets applied to the specific database. If the database field is left blank, the role gets applied to the database instance.</li><li>Role – The user is added to this role</li><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li><li>Paired Action's Name — (Optional) Edit the name of the paired action</li></ul>                                                                                                                                                        |
| Create Windows File Share        | Creates a temporary, encrypted SMB share on the activity's target Windows resource. This action creates a hidden lock file at the root of the share's map path. This is used by the console to identify a folder as being created/owned by the console. Users should **not** attempt to modify or delete this lock file. | <ul><li>Windows File Share Name — The name of the File Share that will be created on the activity's target Windows resource</li><li>Windows File Share Map Path — A folder will be created in this location (local path) as a map for the session's Windows File Share</li><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li><li>Paired Action's Name — (Optional) Edit the name of the paired action</li></ul>                                                                                                                              |
| Enable RDP on Host               | Allows the user to enable RDP on the Host during the pre-session. This action comes paired with the Disable RDP on Host post-session action type.                                                                                                                                                                        | <ul><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li><li>Paired Action's Name — (Optional) Edit the name of the paired action</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                     |
| Monitor File for Changes         | Monitors the specified file for modifications. If no changes are detected, an _Info_ event is logged to the action log. If changes are detected (or the file goes missing after the pre-session hash), a _Warning_ is logged to the action and event logs.                                                               | <ul><li>File to Monitor — Enter the full local path to the file to be monitored</li><li>Fail Pre-Session on Missing File — Fail session provisioning if the file to monitor cannot be found</li><li>Fail Post-Session on Mismatch — Fail session during Activity post-session if monitored file checksum does not match</li><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li><li>Paired Action's Name — (Optional) Edit the name of the paired action</li></ul>                                                                             |
| Purge Kerberos Tickets for Host  | Purge all Kerberos tickets from the resource                                                                                                                                                                                                                                                                             | <ul><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| Purge Kerberos Tickets for User  | Purge Kerberos Tickets for the login account                                                                                                                                                                                                                                                                             | <ul><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| Run AD Replication for User      | Runs AD replication for the target user within the AD site of the target resource                                                                                                                                                                                                                                        | <ul><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| Run Custom PowerShell Script     | Execute custom PowerShell script during Activity                                                                                                                                                                                                                                                                         | <ul><li>Definition — Enter a custom Powershell script here</li><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                       |
| Run Replication                  | Run replication (via repadmin /syncall) on closest domain controller to the target resource                                                                                                                                                                                                                              | <ul><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| Scan Host for Local Groups       | Scan local groups on the resource                                                                                                                                                                                                                                                                                        | <ul><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| Set Cisco User Privilege         | Set Cisco user Privilege level for account access (1-15) 1 = lowest rights 15 = highest rights                                                                                                                                                                                                                           | <ul><li>Privilege — User Privilege level for account access (1-15) 1 = lowest rights 15 = highest rights</li><li>Continue on Error — If an error occurs, continue to the next step rather than halting the action</li><li>Action Name — (Optional) Edit the name of the action</li></ul>                                                                                                                                                                                                                                                                                                                                                                         |
